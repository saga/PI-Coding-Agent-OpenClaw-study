## 二、文档管理与搜索（SuperSearch 深化）

### 2.1 文档生命周期管理

```
文档生命周期：
  草稿（Draft）
      │ 作者提交审核
      ▼
  审核中（Under Review）
      │ 合规/主管审批
      ▼
  已发布（Published）
      │ 自动入库 SuperSearch
      ▼
  已归档（Archived）        ← 超过有效期或被新版本替代
      │
      ▼
  已废止（Superseded）      ← 明确被新报告取代，搜索时降权
```

**关键元数据字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| doc_id | UUID | 唯一标识 |
| title | string | 报告标题 |
| author | string[] | 作者列表 |
| team | enum | 股票/固收/宏观/量化/ESG |
| asset_class | enum[] | 覆盖资产类别 |
| region | enum[] | 覆盖地区 |
| tickers | string[] | 相关股票代码 |
| publish_date | date | 发布日期 |
| valid_until | date | 有效期（可选） |
| access_level | enum | PUBLIC/TEAM/RESTRICTED |
| version | int | 版本号 |
| superseded_by | UUID | 被哪个文档取代 |
| tags | string[] | 自由标签 |

### 2.2 权限控制（Chinese Wall）完整方案

金融机构的信息壁垒（Chinese Wall）是监管要求，不是可选项。

```
权限层级设计：

Level 1：公开（PUBLIC）
  → 所有员工可见
  → 例：已发布的市场评论、宏观展望

Level 2：团队内（TEAM）
  → 仅本团队可见
  → 例：股票研究团队的内部评级

Level 3：限制（RESTRICTED）
  → 需要明确授权
  → 例：并购相关研究、未发布的重大信息

Level 4：隔离（WALL）
  → 严格信息壁垒，跨团队访问需合规审批
  → 例：投行业务相关信息（如适用）
```

**技术实现**：

```
查询时的权限过滤（不可绕过）：

用户发起查询
    │
    ▼
[身份验证] → 获取 user_id, team_id, access_groups
    │
    ▼
[权限过滤器注入]
  OpenSearch 查询自动附加：
  {
    "filter": {
      "terms": {
        "access_level": user.allowed_levels
      }
    }
  }
    │
    ▼
[结果返回] → 用户只能看到有权限的文档
    │
    ▼
[审计日志] → 记录所有查询（用于合规审计）
```

### 2.3 版本管理与变更追踪

- 每次文档更新自动创建新版本，保留历史版本
- 版本对比功能：高亮显示两个版本之间的差异
- 观点变更追踪：自动检测同一分析师对同一公司观点的变化（NLP 驱动）

## 三、量化研究支持模块（核心新增）

### 3.1 研究报告 → 量化信号的转化流程

这是基本面研究与量化研究融合的关键桥梁：

```
┌─────────────────────────────────────────────────────────────┐
│              基本面研究 → 量化信号 转化流程                    │
│                                                             │
│  研究报告（PDF/Word）                                         │
│       │                                                     │
│       ▼                                                     │
│  [NLP 信号提取器]                                            │
│  ├── 情感分析：正面/负面/中性                                  │
│  ├── 目标价提取：analyst_target_price                        │
│  ├── 评级提取：Buy/Hold/Sell + 变化方向                       │
│  ├── 关键指标提取：EPS 预期、收入预期、利润率预期               │
│  └── 风险因素提取：主要风险关键词                              │
│       │                                                     │
│       ▼                                                     │
│  [结构化信号数据库]（PostgreSQL）                              │
│  ├── signal_id, ticker, analyst, date                       │
│  ├── sentiment_score: -1.0 ~ +1.0                          │
│  ├── rating: BUY/HOLD/SELL                                  │
│  ├── target_price, upside_pct                               │
│  └── key_risks[], key_catalysts[]                           │
│       │                                                     │
│       ▼                                                     │
│  [量化因子计算]                                               │
│  ├── 分析师情绪因子：聚合多个分析师的情绪评分                   │
│  ├── 评级变化因子：评级上调/下调的动量信号                      │
│  └── 目标价偏离因子：当前价格 vs 分析师目标价的偏离度            │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 研究知识图谱

将研究报告中的实体关系结构化，构建公司/行业/分析师关系网络：

```
知识图谱节点：
├── 公司节点（Company）：ticker, name, sector, country
├── 分析师节点（Analyst）：name, team, coverage
├── 行业节点（Sector）：GICS 分类
├── 主题节点（Theme）：AI、ESG、利率敏感等
└── 事件节点（Event）：财报、并购、政策变化

知识图谱边：
├── 分析师 → 覆盖 → 公司（COVERS）
├── 公司 → 属于 → 行业（BELONGS_TO）
├── 公司 → 供应商 → 公司（SUPPLIER_OF）
├── 报告 → 提及 → 公司（MENTIONS）
└── 报告 → 关联 → 主题（RELATED_TO）
```

**应用场景**：
- "找所有覆盖台积电的分析师的最新观点" → 图谱查询
- "台积电的主要客户有哪些，他们的研究报告怎么说" → 供应链传导分析
- "AI 主题下，哪些公司被最多分析师正面提及" → 主题热度分析

### 3.4 事件驱动研究与量化信号联动

```
财报发布事件
    │
    ├── 触发：自动抓取财报文本
    │         NLP 提取关键数字和管理层语调
    │         与分析师预期对比（Beat/Miss/In-line）
    │
    ├── 触发：更新公司知识图谱
    │         更新分析师预期数据库
    │
    └── 触发：量化信号更新
              财报超预期因子重新计算
              管理层情绪因子更新
              推送给量化研究员
```

```
示例：某量化因子的血缘链

```
新数据源评估流程（标准化）：

### 7.2 分析师预期 vs 量化模型的分歧信号

**分歧信号是高价值 Alpha 来源**：

```python
# 分歧信号计算示例
def compute_divergence_signal(ticker: str, date: str) -> float:
    """
    计算基本面分析师观点与量化模型的分歧程度
    分歧大 → 可能存在信息不对称 → 潜在 Alpha 机会
    """
    # 分析师综合情感评分（-1 到 +1）
    analyst_score = get_analyst_sentiment(ticker, date)

    # 量化模型综合评分（-1 到 +1）
    quant_score = get_quant_composite_score(ticker, date)

    # 分歧度
    divergence = analyst_score - quant_score

    # 分歧信号解读：
    # divergence > 0.5：分析师比量化模型更乐观 → 可能有基本面催化剂未被量化捕捉
    # divergence < -0.5：量化模型比分析师更乐观 → 可能有技术面信号未被分析师关注
    return divergence
```

### 7.3 事件研究（Event Study）框架

将基本面事件与量化收益分析结合：

```
事件类型：
├── 财报超预期（Earnings Beat）
├── 分析师评级上调（Rating Upgrade）
├── 管理层变更（Management Change）
├── 并购公告（M&A Announcement）
└── ESG 争议事件（ESG Controversy）

事件研究输出：
├── 事件前后的累计超额收益（CAR）
├── 不同市场环境下的事件效应差异
├── 事件效应的持续时间（信号有效期）
└── 与其他因子的交互效应
```

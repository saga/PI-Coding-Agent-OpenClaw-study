> 日期：2026-03-14
> 覆盖：AI Agent 量化应用、大模型金融微调（FinLLM）、量化 ESG 投资、加密资产量化研究

## 一、AI Agent 在量化投资中的应用

### 1.1 从 LLM 工具到 Agentic 系统的演进

```
第一阶段（2022-2023）：LLM 作为工具
  → 用 ChatGPT 辅助写代码、解读财报
  → 被动响应，无自主规划能力

第二阶段（2024）：LLM + RAG
  → 基于内部知识库的问答
  → 仍是单轮交互，无持续任务执行

第三阶段（2025-至今）：Agentic AI
  → 自主规划、执行、反思、迭代
  → 多 Agent 协作，覆盖完整研究流程
```

### 1.2 自主因子发现（Alpha-GPT 范式深化）

**核心架构**：ReAct Loop（推理 + 行动）

```
[研究员设定目标]
      ↓
[Agent 分解任务]
      ↓
[数据获取 Agent] → 拉取价格、财务、另类数据
      ↓
[假设生成 Agent] → LLM 生成因子逻辑（自然语言）
      ↓
[代码生成 Agent] → 将逻辑转化为 Python 因子代码
      ↓
[回测 Agent]     → 执行回测，计算 IC/ICIR/分组收益
      ↓
[评估 Agent]     → 判断是否通过质量门槛
      ↓
[迭代/上线]      → 反馈给假设生成 Agent 或提交审核
```

**2025 年前沿进展**：
- **R&D-Agent-Quant**（NeurIPS 2025）：多 Agent 框架自动化量化策略开发，用 70% 更少的因子实现 2x 年化收益
- **QuantEvolve**：多 Agent 进化框架，维护多样化策略集合，适应不同市场状态
- **AlphaAgents**：角色化多 Agent 系统，基本面/情绪/估值 Agent 协作，通过辩论协议减少幻觉

### 1.3 多 Agent 协作研究框架

**典型角色分工**：

| Agent 角色 | 职责 | 工具 |
|-----------|------|------|
| 数据 Agent | 获取、清洗、对齐数据 | SQL、API、数据湖 |
| 研究 Agent | 生成投资假设、因子逻辑 | LLM + 知识库 RAG |
| 代码 Agent | 将假设转化为可执行代码 | Code Interpreter |
| 回测 Agent | 执行回测、计算绩效指标 | 回测引擎 API |
| 风控 Agent | 检查过拟合、容量、合规 | 统计检验工具 |
| 报告 Agent | 生成研究报告、可视化 | 图表库、模板引擎 |

**协作模式**：
- **串行流水线**：适合标准化因子研究流程
- **辩论协议**（Debate Protocol）：多个 Agent 对同一假设提出支持/反对意见，减少单点偏差
- **自我进化**（Self-Evolving）：Agent 积累"技能记忆"和"经验记忆"，持续改进

### 1.4 LLM 辅助代码生成（量化研究员的 Copilot）

**应用场景**：
- 自然语言描述因子逻辑 → 自动生成 Python/SQL 代码
- 代码审查：检测前视偏差、数据泄露
- 文档生成：自动为因子代码生成注释和说明
- 调试辅助：解释回测异常，定位 bug

**与通用 Copilot 的区别**：
- 需要金融领域知识（因子命名规范、回测框架 API）
- 需要理解 Point-in-Time 数据约束
- 需要内置合规检查（禁止使用未来数据）

### 1.5 对 Fidelity 的建议

- 短期：将 LLM Copilot 集成到 JupyterHub，辅助因子代码生成
- 中期：构建 Agentic 因子发现流水线，人工审核作为最终门槛
- 长期：多 Agent 协作研究框架，覆盖从数据到报告的完整链路
- 风险控制：所有 Agent 生成的因子必须经过严格的样本外验证

## 二、大模型金融微调（FinLLM）

### 2.1 为什么需要金融专用 LLM

通用 LLM（GPT-4、Claude）在金融任务上的局限：
- 不理解金融术语的精确含义（如"做空"、"久期"、"凸性"）
- 对财报数字的理解不够精准
- 缺乏对监管文件（10-K、招股书）的深度理解
- 幻觉问题在金融场景中风险极高

### 2.2 BloombergGPT

**基本信息**：
- 参数量：500 亿（50B）
- 架构：Decoder-only Transformer，70 层，ALiBi 位置编码
- 训练数据：3630 亿 token（Bloomberg 专有金融数据 + 通用数据混合）
- 发布时间：2023 年

**训练数据构成**：

| 数据类型 | 规模 | 来源 |
|---------|------|------|
| 金融新闻 | 大量 | Bloomberg 新闻档案 |
| 财务报告 | 大量 | SEC 文件、年报 |
| 社交媒体 | 适量 | 金融相关推文 |
| 通用文本 | 混合 | The Pile 数据集 |

**优势**：在金融 NLP 基准测试（FPB、FiQA、NER）上显著优于同规模通用模型

**局限**：闭源，仅供 Bloomberg 内部使用；参数量大，推理成本高

### 2.3 FinBERT 系列模型

**FinBERT（原版）**：
- 基于 BERT-base，在金融文本上继续预训练
- 核心任务：金融情绪分析（正面/负面/中性）
- 广泛用于：新闻情绪因子、财报语调分析

**FinBERT2（2025 年）**：
- 在 320 亿 token 高质量金融语料上预训练
- 在 5 个金融分类任务上比 FinBERT 提升 0.4%-3.3%
- 比 GPT-4 等大模型在分类任务上高出 9.7%-12.3%
- 优势：小模型、低推理成本、高精度

**FinBERT 在量化中的应用**：
```python
# 典型应用：财报电话会议情绪因子
from transformers import pipeline

sentiment_pipeline = pipeline("sentiment-analysis", model="ProsusAI/finbert")

# 对财报文本打分
earnings_text = "Revenue exceeded expectations, margins improved significantly..."
result = sentiment_pipeline(earnings_text)
# → {'label': 'positive', 'score': 0.94}

# 构建情绪因子：正面情绪得分 - 负面情绪得分
```

### 2.4 FinGPT 与开源生态

**FinGPT**（开源）：
- 基于 LLaMA/Mistral 微调，完全开源
- 支持情绪分析、股价预测、财报摘要等任务
- 社区活跃，持续更新

**Open-FinLLMs 框架**：
- 统一评估多个金融 LLM 的基准框架
- 覆盖：情绪分析、关系抽取、问答、预测

### 2.5 自建金融 LLM 的可行性评估

**适合自建的场景**：
- 有大量内部专有数据（研究报告、交易记录、内部分析）
- 需要严格数据安全（不能将数据发送给第三方 API）
- 有特定的金融子领域需求（如 Fidelity 的亚太市场研究）

**建议路径（Fidelity）**：

| 阶段 | 方案 | 成本 | 效果 |
|------|------|------|------|
| 短期 | 调用 GPT-4/Claude API + RAG | 低 | 快速上线 |
| 中期 | FinBERT2 微调（分类任务） | 中 | 高精度、低成本 |
| 长期 | 基于 LLaMA 的 Fidelity-LLM | 高 | 完全定制、数据安全 |

**关键考量**：
- 训练数据质量 > 模型规模
- 金融幻觉（Hallucination）风险需要专门的事实核查机制
- 监管合规：模型输出需要可解释、可审计

## 三、量化 ESG 投资

### 3.1 ESG 因子构建方法论

**ESG 数据来源**：

| 来源类型 | 代表供应商 | 特点 |
|---------|-----------|------|
| 第三方评级 | MSCI ESG、Sustainalytics、ISS | 覆盖广，但评级分歧大 |
| 监管披露 | SEC、CSRD（欧盟）、港交所 | 标准化，但滞后 |
| NLP 提取 | 从年报/CSR 报告提取 | 及时，但噪声大 |
| 卫星/另类 | 碳排放卫星数据、供应链数据 | 客观，但覆盖有限 |

**2025 年重要变化**：欧盟 CSRD（企业可持续发展报告指令）于 2025 年 3 月要求约 5000 家大型欧盟企业首次披露 FY2024 ESG 数据，为量化研究提供了新的标准化数据源。

**ESG 因子构建步骤**：
1. **数据标准化**：不同供应商评级差异巨大，需要 Z-score 标准化
2. **行业中性化**：ESG 评分有强烈的行业偏差（科技 > 能源），需行业内排名
3. **时序对齐**：ESG 数据更新频率低（年度），需要插值处理
4. **信号合成**：E/S/G 三个维度的加权合成（权重因行业而异）

### 3.2 ESG 数据质量问题

**核心挑战**：
- **评级分歧**：同一公司在不同供应商的 ESG 评级相关性仅约 0.5（远低于信用评级的 0.9+）
- **历史数据短**：大多数 ESG 数据历史不超过 10 年，回测可靠性存疑
- **回溯修订**：供应商经常追溯修改历史评级，造成前视偏差
- **覆盖率不均**：大市值公司覆盖好，小市值/新兴市场覆盖差
- **58% 的机构投资者**将 ESG 数据质量差列为最大障碍（2025 年调研）

**应对策略**：
- 多供应商数据融合，降低单一来源偏差
- 使用 NLP 从原始披露文件提取数据，绕过第三方评级
- 建立 Point-in-Time ESG 数据库，避免回溯修订问题

### 3.3 ESG 与传统因子的融合策略

**三种融合模式**：

**模式一：ESG 作为约束条件**
```
传统多因子优化
+ ESG 评分 ≥ 行业中位数（约束）
→ 在 ESG 合规前提下最大化 Alpha
```

**模式二：ESG 作为独立因子**
```
Alpha = w1×价值 + w2×动量 + w3×质量 + w4×ESG改善动量
（ESG 改善动量 = ESG 评分变化率，而非绝对水平）
```

**模式三：ESG 风险调整**
```
风险模型中加入 ESG 风险因子
→ 高 ESG 风险公司获得更高的特异性风险估计
→ 组合优化自然降低 ESG 风险暴露
```

**实证发现**：
- ESG 绝对水平因子 Alpha 有限（已被定价）
- **ESG 改善动量**（评级上调趋势）有更强的预测力
- CSRD 等新监管披露创造了短期数据套利窗口（约 12 个月）

### 3.4 对 Fidelity 的建议

- Fidelity 作为全球资管机构，ESG 整合是监管和客户双重要求
- 建议构建内部 ESG 数据管道：CSRD 披露 → NLP 提取 → 标准化因子
- 重点关注 ESG 改善动量因子，而非静态评级
- 亚太市场 ESG 数据覆盖差，需要专项数据采购策略

## 四、加密资产量化研究

### 4.1 链上数据（On-chain Data）作为另类因子

**链上数据的独特性**：与传统金融数据不同，区块链数据完全公开、实时、不可篡改。

**核心链上指标及其量化含义**：

| 指标 | 含义 | 量化应用 |
|------|------|---------|
| 活跃地址数 | 网络使用率 | 网络价值因子 |
| 交易量（链上） | 真实经济活动 | 区分投机与实用 |
| 大户持仓变化（Whale） | 聪明钱动向 | 跟随大户信号 |
| 交易所净流入/流出 | 抛压/囤币意愿 | 短期价格压力 |
| 矿工持仓 | 供给侧压力 | 中期趋势信号 |
| SOPR（已实现利润率） | 持有者盈亏状态 | 市场情绪指标 |
| NVT Ratio | 网络价值/交易量 | 估值因子 |

**数据来源**：Glassnode、Nansen、Dune Analytics、DefiLlama

### 4.2 DeFi 协议数据

**DeFi 量化信号**：
- **TVL（总锁仓量）变化**：协议吸引力的代理指标
- **借贷利率套利**：跨协议利率差异创造套利机会
- **流动性挖矿收益**：APY 变化预测资金流向
- **清算数据**：大规模清算事件的价格冲击预测

**2025 年 DeFi 规模**：
- 链上经济总规模达 200 亿美元（1124 个协议实现盈利）
- DEX 月交易量突破 2000 亿美元
- 永续合约 DEX 年增长 300%+（Hyperliquid、dYdX 领跑）

### 4.3 加密市场微观结构

**与传统市场的关键差异**：

| 维度 | 传统市场 | 加密市场 |
|------|---------|---------|
| 交易时间 | 有限（交易所时间） | 24/7/365 |
| 做市商 | 受监管的专业机构 | 任何人（AMM） |
| 清算机制 | T+2 结算 | 即时链上结算 |
| 价格发现 | 集中式交易所 | CEX + DEX 并行 |
| 数据透明度 | 有限 | 完全公开（链上） |

**量化策略特有机会**：
- **跨交易所套利**：CEX 与 DEX 之间的价差
- **资金费率套利**：永续合约资金费率的均值回归
- **MEV（最大可提取价值）**：区块链特有的套利机会

### 4.4 对 Fidelity 的建议

- 加密资产量化研究对 Fidelity 当前业务优先级较低，但值得跟踪
- 链上数据分析方法论可迁移到传统资产（如供应链数据分析）
- 建议设立小型研究团队，跟踪加密量化前沿，为未来布局做准备
- 重点关注机构级加密产品（比特币 ETF 套利、稳定币收益策略）

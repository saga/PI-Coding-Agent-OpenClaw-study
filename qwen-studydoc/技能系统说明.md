# Pi 编码助手技能系统说明

## 概述

本文档提供了 pi 编码助手技能系统的详细说明，包括技能发现、注册、扩展和执行流程。

## 1. 技能系统架构

### 1.1 核心组件

pi 编码助手的技能系统由以下三个核心组件组成：

1. **技能发现** (`src/core/skills.ts`)
   - 扫描目录中的 `SKILL.md` 文件
   - 从 YAML frontmatter 中提取元数据（名称、描述）
   - 根据 Agent Skills 规范进行验证
   - 返回包含 `baseDir` 和 `filePath` 的 `Skill` 对象

2. **技能注册** (`src/modes/interactive/interactive-mode.ts`)
   - 将技能转换为斜杠命令 (`/skill:name`)
   - 存储映射：`commandName -> skill.filePath`
   - 注册到自动补全提供者

3. **技能扩展** (`src/core/agent-session.ts`)
   - 将 `/skill:name args` 扩展为完整的技能内容
   - 将技能指令注入到 agent 上下文中
   - 提供 `baseDir` 引用用于相对路径

### 1.2 技能数据结构

```typescript
interface Skill {
  name: string;              // 来自 frontmatter 或目录名
  description: string;       // 必需，用于发现
  filePath: string;          // SKILL.md 的绝对路径
  baseDir: string;           // SKILL.md 的父目录
  source: string;            // "user"、"project" 或 "path"
  disableModelInvocation: boolean;
}
```

## 2. 技能发现流程

### 2.1 发现位置

技能从以下位置发现：

- **全局**：
  - `~/.pi/agent/skills/`
  - `~/.agents/skills/`
- **项目**：
  - `.pi/skills/`
  - `.agents/skills/`（在 `cwd` 和祖先目录中，最多到 git 仓库根或文件系统根）
- **包**：`skills/` 目录或 `package.json` 中的 `pi.skills` 条目
- **设置**：`skills` 数组中的文件或目录
- **CLI**：`--skill <path>`（可重复，即使使用 `--no-skills` 也添加）

### 2.2 发现规则

- 技能目录根中的直接 `.md` 文件
- 子目录下递归的 `SKILL.md` 文件

### 2.3 验证

技能名称必须：
- 1-64 个字符
- 小写字母、数字、连字符
- 不能以连字符开头或结尾
- 不能包含连续连字符
- 必须匹配父目录名

## 3. 技能扩展流程

### 3.1 命令解析

当通过 `/skill:name args` 调用技能时：

```typescript
private _expandSkillCommand(text: string): string {
  if (!text.startsWith("/skill:")) return text;
  
  const spaceIndex = text.indexOf(" ");
  const skillName = spaceIndex === -1 ? text.slice(7) : text.slice(7, spaceIndex);
  const args = spaceIndex === -1 ? "" : text.slice(spaceIndex + 1).trim();
  
  const skill = this.resourceLoader.getSkills().skills.find((s) => s.name === skillName);
  if (!skill) return text;
  
  try {
    const content = readFileSync(skill.filePath, "utf-8");
    const body = stripFrontmatter(content).trim();
    const skillBlock = `<skill name="${skill.name}" location="${skill.filePath}">\nReferences are relative to ${skill.baseDir}.\n\n${body}\n</skill>`;
    return args ? `${skillBlock}\n\n${args}` : skillBlock;
  } catch (err) {
    // 错误处理...
    return text;
  }
}
```

### 3.2 技能块注入

1. 读取完整的 `SKILL.md` 内容
2. 去除 YAML frontmatter
3. 包装在 `<skill>` XML 块中
4. 注入 `baseDir` 引用："References are relative to {baseDir}"
5. 在技能内容后追加用户参数

### 3.3 Agent 处理

1. 将扩展的技能块作为用户消息接收
2. Agent LLM 读取技能指令
3. LLM 使用工具执行任务（包括 `bash` 工具）

## 4. 技能命令

技能注册为 `/skill:name` 命令：

```bash
/skill:brave-search           # 加载并执行技能
/skill:pdf-tools extract      # 带参数加载技能
```

在交互模式下，可以通过 `/settings` 或 `settings.json` 切换技能命令：

```json
{
  "enableSkillCommands": true
}
```

## 5. 技能结构

技能是带有 `SKILL.md` 文件的目录。其他内容是自由格式的。

```
my-skill/
├── SKILL.md              # 必需：frontmatter + 指令
├── scripts/              # 辅助脚本
│   └── process.sh
├── references/           # 按需加载的详细文档
│   └── api-reference.md
└── assets/
    └── template.json
```

### SKILL.md 格式

```markdown
---
name: my-skill
description: 这个技能做什么以及何时使用。要具体。
---

# 我的技能

## 设置

首次使用前运行一次：
```bash
cd /path/to/skill && npm install
```

## 使用方法

```bash
./scripts/process.sh <input>
```
```

## 6. 技能上下文

技能的 `baseDir` 是其 `SKILL.md` 文件的父目录。所有相对路径都相对于此目录解析。

**重要**：当前实现中，技能的 `baseDir` 在提示中被提及，但没有被强制执行。这导致了脚本路径解析问题。

## 7. 当前问题

### 7.1 问题描述

技能引用的脚本使用相对路径（如 `scripts/myscript.py`），但 bash 工具在 `process.cwd()`（项目根目录）执行，而不是技能的 `baseDir`。

### 7.2 根本原因

1. **Bash 工具设计**：`bash` 工具接受 `cwd` 参数，但技能没有指定它
2. **Agent 上下文**：技能作为文本注入，而不是具有自己上下文的可执行单元
3. **LLM 行为**：LLM 读取技能指令，但不知道正确的 working directory
4. **缺少技能上下文**：技能的 `baseDir` 在提示中被提及，但没有被强制执行

## 8. 解决方案

### 8.1 推荐方案：混合方案

结合提示工程和智能默认值：

1. **增强技能块**：在技能 XML 块中包含显式的路径解析规则
2. **技能上下文跟踪**：在技能扩展期间跟踪当前技能的 `baseDir`
3. **智能 Bash 工具**：检查技能上下文并在执行相对路径时使用技能的 `baseDir`

### 8.2 实现步骤

1. 修改 `_expandSkillCommand` 以包含 `baseDir` 和路径解析规则
2. 在 `AgentSession` 中添加 `currentSkillContext` 字段
3. 修改 `createBashTool` 以检查技能上下文
4. 在技能扩展时设置技能上下文
5. 在 agent 轮次后重置技能上下文

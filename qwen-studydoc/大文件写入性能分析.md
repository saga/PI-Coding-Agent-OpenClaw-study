# 大文件写入性能分析

## 执行摘要

本文档分析了为什么写入大文件时 PI 编码助手会变慢并反复重试，使用 Claude Sonnet 4.6 作为示例。

## 1. 问题描述

### 1.1 用户报告的现象

- 写入稍微大一点的文件时，PI 变得很慢
- 可能反复重试
- 使用 Claude Sonnet 4.6

### 1.2 可能的原因

1. **工具执行超时**：写入大文件需要时间，可能触发超时
2. **重试机制**：错误被误判为可重试错误
3. **上下文窗口**：大文件内容可能接近上下文窗口限制
4. **LLM 响应时间**：大文件导致 LLM 处理时间变长

```typescript
async function fetchWithRetry(url: string, options: RequestInit, maxRetries: number) {
  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    try {
      const response = await fetch(url, options);
      
      if (!response.ok) {
        const errorText = await response.text();
        
        // 检查是否可重试
        if (attempt < maxRetries && isRetryableError(response.status, errorText)) {
          const serverDelay = extractRetryDelay(errorText, response);
          
          if (serverDelay !== undefined) {
            const maxDelayMs = options?.maxRetryDelayMs ?? 60000;
            const delay = Math.min(serverDelay, maxDelayMs);
            
            if (delay > 0) {
              await sleep(delay);
            }
          }
        } else {
          // 不可重试或最大重试次数已用完
          throw new Error(`API error (${response.status}): ${errorText}`);
        }
      }
    } catch (err) {
      // 网络错误是可重试的
      if (attempt < maxRetries && isNetworkError(err)) {
        continue;
      }
      throw err;
    }
  }
}
```

- **3 次重试**：默认最大重试次数
- **指数退避**：根据服务器请求的延迟
- **网络错误重试**：网络错误会被重试

## 3. 根本原因分析

### 3.1 问题 1：工具执行无超时

**症状**：写入大文件时，工具执行时间变长，但没有超时限制

**原因**：
1. `write` 和 `edit` 工具没有内置超时
2. 仅依赖 `AbortSignal` 处理取消
3. 如果 LLM 请求超时，工具可能仍在后台执行

**影响**：
- 大文件写入可能需要数秒
- 没有超时会导致等待时间不确定
- 可能与重试机制冲突

### 3.2 问题 2：重试机制误判

**症状**：写入失败被重试，但实际是不可重试的错误

**可能的错误模式**：
1. **文件系统错误**：权限问题、磁盘空间不足
2. **路径错误**：无效路径、目录不存在
3. **内容太大**：写入内容超过系统限制

**重试逻辑**：
```typescript
// 可重试错误检测
return /overloaded|rate.?limit|too many requests|429|500|502|503|504|service.?unavailable|server error|internal error|connection.?error|connection.?refused|other side closed|fetch failed|upstream.?connect|reset before headers|terminated|retry delay/i.test(err);
```

**问题**：
- 不匹配的错误不会被重试
- 匹配的错误会被重试（最多 3 次，2s+4s+8s = 14s）
- 如果错误是永久性的，重试是浪费时间

### 3.3 问题 3：上下文窗口限制

**症状**：大文件内容可能导致上下文窗口溢出

**原因**：
1. 写入的文件内容会添加到对话历史
2. Claude Sonnet 4.6 上下文窗口有限
3. 大文件可能导致上下文溢出

**影响**：
- 上下文溢出不会被重试（由压缩处理）
- 但可能导致 LLM 响应变慢
- 可能触发其他错误

### 3.4 问题 4：编辑工具的读写开销

**症状**：编辑工具比写入工具更慢

**原因**：
1. **读取整个文件**：`fs.readFile` 读取完整文件内容
2. **内容处理**：模糊匹配、规范化、替换
3. **写入整个文件**：`fs.writeFile` 写入修改后的内容

**影响**：
- 大文件需要完整加载到内存
- 处理时间随文件大小线性增长
- 双倍 I/O（读 + 写）

## 4. 性能瓶颈分析

### 4.1 写入工具性能

**操作流程**：
1. 解析路径（毫秒级）
2. 创建父目录（毫秒级）
3. 写入文件（取决于文件大小）
   - 小文件 (<1MB)：10-50ms
   - 中等文件 (1-10MB)：50-500ms
   - 大文件 (10-100MB)：500ms-5s
   - 超大文件 (>100MB)：5s+

**瓶颈**：
- **I/O 写入**：磁盘写入速度
- **内存分配**：大字符串分配
- **UTF-8 编码**：内容编码

### 4.2 编辑工具性能

**操作流程**：
1. 读取整个文件（取决于文件大小）
2. 解析内容（毫秒级）
3. 模糊匹配（取决于文件大小）
4. 替换内容（取决于文件大小）
5. 写入修改后的内容（取决于文件大小）

**瓶颈**：
- **读取时间**：与文件大小成正比
- **匹配时间**：O(n*m) 复杂度
- **写入时间**：与文件大小成正比

### 4.3 重试延迟

**默认配置**：
- 第 1 次重试：2s 后
- 第 2 次重试：4s 后
- 第 3 次重试：8s 后
- **总延迟**：14s（如果 3 次都失败）

**指数退避**：
```
delay = baseDelayMs * 2^(attempt-1)

attempt 1: 2000ms (2s)
attempt 2: 4000ms (4s)
attempt 3: 8000ms (8s)
total:   14000ms (14s)
```

```typescript
// 添加超时选项
export interface WriteToolOptions {
  /** Custom operations for file writing. Default: local filesystem */
  operations?: WriteOperations;
  /** Timeout in milliseconds. Default: 30000 (30 seconds) */
  timeout?: number;
}

export function createWriteTool(cwd: string, options?: WriteToolOptions): AgentTool {
  const ops = options?.operations ?? defaultWriteOperations;
  const timeoutMs = options?.timeout ?? 30000;  // 默认 30 秒
  
  return {
    name: "write",
    label: "write",
    description: "Write content to a file. Creates the file if it doesn't exist, overwrites if it does. Automatically creates parent directories.",
    parameters: writeSchema,
    execute: async (
      _toolCallId: string,
      { path, content }: { path: string; content: string },
      signal?: AbortSignal,
    ) => {
      const absolutePath = resolveToCwd(path, cwd);
      const dir = dirname(absolutePath);

      return new Promise<{ content: Array<{ type: "text"; text: string }>; details: undefined }>(
        (resolve, reject) => {
          // 检查是否已中止
          if (signal?.aborted) {
            reject(new Error("Operation aborted"));
            return;
          }

          let aborted = false;

          // 设置中止处理程序
          const onAbort = () => {
            aborted = true;
            reject(new Error("Operation aborted"));
          };

          if (signal) {
            signal.addEventListener("abort", onAbort, { once: true });
          }

          // 设置超时
          const timeoutId = setTimeout(() => {
            aborted = true;
            if (signal) {
              signal.removeEventListener("abort", onAbort);
            }
            reject(new Error(`Write operation timed out after ${timeoutMs}ms`));
          }, timeoutMs);

          // 执行写入操作
          (async () => {
            try {
              // 创建父目录
              await ops.mkdir(dir);

              // 检查是否已中止
              if (aborted) {
                clearTimeout(timeoutId);
                return;
              }

              // 写入文件
              await ops.writeFile(absolutePath, content);

              // 检查是否已中止
              if (aborted) {
                clearTimeout(timeoutId);
                return;
              }

              // 清除超时和中止处理程序
              clearTimeout(timeoutId);
              if (signal) {
                signal.removeEventListener("abort", onAbort);
              }

              resolve({
                content: [{ type: "text", text: `Successfully wrote ${content.length} bytes to ${path}` }],
                details: undefined,
              });
            } catch (error: any) {
              // 清除超时和中止处理程序
              clearTimeout(timeoutId);
              if (signal) {
                signal.removeEventListener("abort", onAbort);
              }

              if (!aborted) {
                reject(error);
              }
            }
          })();
        },
      );
    },
  };
}
```

```typescript
import { createWriteStream } from "fs";
import { pipeline } from "stream/promises";

export async function writeLargeFile(path: string, content: string): Promise<void> {
  const writeStream = createWriteStream(path, { encoding: "utf-8" });
  
  // 分块写入
  const chunkSize = 1024 * 1024; // 1MB
  for (let i = 0; i < content.length; i += chunkSize) {
    const chunk = content.substring(i, i + chunkSize);
    writeStream.write(chunk);
    
    // 检查是否需要暂停
    if (!writeStream.write("")) {
      await new Promise((resolve) => writeStream.once("drain", resolve));
    }
  }
  
  writeStream.end();
  await new Promise((resolve) => writeStream.on("finish", resolve));
}

        return writeLargeFile(absolutePath, content);
      }
      
      // 小文件使用传统写入
      await ops.writeFile(absolutePath, content);
```

```typescript
  if (err.includes("500") || err.includes("503")) return true;
  if (err.includes("service unavailable")) return true;
  if (err.includes("connection error")) return true;
  
  // 不可重试错误
  if (err.includes("permission")) return false;
  if (err.includes("not found")) return false;
  if (err.includes("invalid")) return false;
  if (err.includes("file not found")) return false;
  if (err.includes("access denied")) return false;
  
  // 默认：不重试
  return false;
}
```

### 6.2 配置参数说明

#### 6.2.1 retry 配置

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| enabled | boolean | true | 是否启用重试 |
| maxRetries | number | 3 | 最大重试次数（建议改为 1） |
| baseDelayMs | number | 2000 | 基础延迟（毫秒） |
| maxDelayMs | number | 60000 | 最大延迟（毫秒） |

**重试延迟计算**：
```
maxRetries: 1 (建议)
├─ 第 1 次尝试（初始）
└─ 失败 → 等 2s → 第 2 次尝试（重试 1 次）
    └─ 再次失败 → 抛出错误
总延迟：2s

maxRetries: 3 (默认)
├─ 第 1 次尝试（初始）
├─ 失败 → 等 2s → 第 2 次尝试（重试 1 次）
├─ 失败 → 等 4s → 第 3 次尝试（重试 2 次）
└─ 失败 → 等 8s → 第 4 次尝试（重试 3 次）
    └─ 再次失败 → 抛出错误
总延迟：14s (2+4+8)
```

**重试次数说明**：
- `maxRetries: 1` = 最多 2 次尝试（1 初始 + 1 重试）
- `maxRetries: 3` = 最多 4 次尝试（1 初始 + 3 重试）
- 所有尝试都失败后，才会抛出错误等待用户处理
- **建议生产环境使用 `maxRetries: 1`**，快速失败并减少等待时间

#### 6.2.2 tools 配置

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| write.timeout | number | 无超时 | 写入工具超时（毫秒） |
| edit.timeout | number | 无超时 | 编辑工具超时（毫秒） |

### 6.5 注意事项

1. **超时设置**：
   - 小文件 (<1MB)：10-30 秒足够
   - 中等文件 (1-10MB)：30-60 秒
   - 大文件 (10-100MB)：60-120 秒
   - 超大文件 (>100MB)：考虑分块写入

2. **重试策略**：
   - `maxRetries: 1` = 最多 2 次尝试（1 初始 + 1 重试），失败后快速抛出错误
   - `maxRetries: 3` = 最多 4 次尝试（1 初始 + 3 重试），总延迟可达 14s
   - 生产环境建议 `maxRetries: 1`，快速失败并减少等待时间
   - 开发环境可以使用 `maxRetries: 2-3`，增加成功概率
   - 如果错误不可重试（如文件权限、路径错误），重试是浪费时间

3. **错误处理**：
   - 捕获超时错误并适当处理
   - 记录重试日志以便调试
   - 所有重试都失败后，错误会抛出给用户处理
   - 考虑实现自定义重试逻辑

4. **监控和调优**：
   - 监控工具执行时间
   - 根据实际文件大小调整超时
   - 根据错误率调整重试次数
